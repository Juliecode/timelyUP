节目内容 / 应用分析

本期专题：给我讲解下最新claude 和codex的进展，功能以及对比

1. Codex是专注于代码生成、调试和优化的AI，能将自然语言转化为嵌入式C、MATLAB等代码，大幅提升编码效率。（来源：专题讲解）

2. Claude是通用的AI技术顾问，擅长深度推理、理解复杂和长篇技术文档，辅助系统架构设计、功能安全分析、前沿技术转化。（来源：专题讲解）

3. Codex可用于EPS的嵌入式C代码生成（FOC、故障诊断）、MATLAB/Simulink脚本辅助、测试用例生成及代码理解与排障。（来源：专题讲解）

4. Claude可协助EPS的线控转向架构设计、ISO 26262功能安全分析、前沿电机控制策略转化、AI/数据驱动方案设计及故障诊断。（来源：专题讲解）

5. 两者相辅相成：Claude负责高层设计与分析，Codex负责具体实现，共同加速EPS电机控制、线控转向等复杂系统的研发进程。（来源：专题讲解）

6. AI大模型是研发工程师的强大助手，而非替代者，赋能工程师聚焦创新和解决核心工程难题。（来源：专题讲解）

完整口播文稿  ·  下载 .txt

各位EPS产品研发工程师朋友们，大家好！我是你们的老朋友，今天想和大家聊聊两个正在改变我们研发范式的AI大模型：Anthropic的Claude和OpenAI的Codex。我知道大家平时都在忙着永磁同步电机的FOC控制、线控转向的架构设计、ISO 26262功能安全这些硬核技术，那这两个AI究竟能给我们带来什么新可能呢？咱们今天就来好好盘盘。目标是七分钟，咱们争取把事情说透。

首先，我们来认识一下它们。Codex，你可以把它想象成一位超级精通编程的同事，它是由OpenAI开发的，基于GPT系列大模型，但它特别擅长把自然语言指令转换成各种编程代码，比如C语言、Python、甚至MATLAB脚本。它不光能写代码，还能帮你调试、解释代码逻辑、重构代码，甚至生成测试用例。GitHub Copilot就是基于类似技术的一个典型应用。简单来说，Codex就是你的代码副驾驶，能极大提升你的编码效率。

而Claude呢，它是Anthropic公司推出的一系列大型语言模型。相比Codex专注于代码，Claude更像是一个全能型的技术顾问，它特别擅长处理复杂的信息、进行深度推理、理解超长的文档内容。它以“有益、无害、诚实”为设计理念，在处理开放性问题、进行逻辑分析和提供洞察方面表现出色。最新的Claude 3系列，无论是Opus、Sonnet还是Haiku，都在通用理解、推理能力和上下文窗口大小上达到了新的高度。你可以把它看作是一个能够深度理解并和你讨论技术细节的资深专家。

那么，为什么它们对我们很重要呢？Codex的出现，正在革新软件开发的流程，它能把我们从大量的重复性编码工作中解放出来，让我们有更多精力去思考系统设计和算法创新。而Claude，则在推动AI的推理和长文本理解能力边界，它能帮助我们处理海量的技术资料，进行更深层次的决策辅助和问题分析。

接下来，咱们就聊聊大家最关心的，它们如何与我们的EPS研发工作结合，有哪些可落地的应用场景。

先说Codex。作为一名EPS工程师，大家每天都要和嵌入式C代码、MATLAB/Simulink模型打交道。Codex在这里能发挥巨大作用：
1. **嵌入式C代码生成与优化：** 比如，你可以用自然语言描述一个FOC控制器的电流环代码逻辑，或者一个故障诊断模块的实现思路，Codex就能帮你生成初步的C代码。它还能帮你审查现有代码，提出优化建议，比如提升执行效率、降低内存占用，或者确保代码符合ISO 26262的功能安全要求，比如添加必要的错误检查和冗余逻辑。
2. **MATLAB/Simulink脚本辅助：** 在模型化设计与代码生成过程中，Codex能帮你编写MATLAB脚本，用于自动化处理标定数据、生成复杂的测试向量、或者进行模型参数的批量调整。甚至可以辅助生成Simulink模型中的S-Function代码。
3. **测试用例与验证：** 基于你的功能需求文档，Codex可以帮你自动生成单元测试用例，或者为MIL/SIL/HIL仿真平台生成测试脚本，大大加速验证流程。
4. **代码理解与排障：** 当你面对一段复杂的、缺乏文档的遗留代码时，Codex能帮你快速理解其功能和逻辑，甚至帮你定位潜在的bug，这对于提升电机控制策略的鲁棒性非常有帮助。

再来看Claude，它更像是你的高级技术参谋：
1. **系统架构与功能安全设计：** 对于线控转向这种高度复杂的系统，Claude能够帮你梳理和分析ISO 26262等功能安全标准，基于你的需求，辅助你设计具备高冗余、高安全性的系统架构。它可以阅读大量的规范文档，帮你提炼关键要求，甚至帮你评估不同架构方案的优劣。
2. **电机控制策略与前沿研究转化：** 你可以把最新的机器人关节执行器控制、电机控制领域的学术论文喂给Claude，让它帮你总结核心算法、分析与PMSM永磁同步电机控制的关联点，并提出如何将其快速转化为可落地的EPS工程方案。比如，如何将新的无传感器控制算法、转矩脉动抑制技术应用到你的项目中。
3. **AI与数据驱动应用：** 在你最想解决的用AI改进控制、标定和故障诊断问题上，Claude可以帮你规划AI模型的设计思路，比如如何构建神经网络观测器、如何设计强化学习控制器的奖励函数，甚至帮你分析海量台架测试数据，找出温漂、NVH问题的潜在模式。
4. **技术文档与知识管理：** 处理大量的技术报告、生成设计文档、或者为团队创建培训材料，Claude都能高效完成，确保信息准确且易于理解。

所以你看，Codex和Claude并非互相替代，而是相辅相成。Codex是你的得力编码助手，专注于“实现”；而Claude是你的智慧参谋，专注于“理解”和“设计”。举个例子，Claude可以帮你深入分析线控转向的冗余架构和功能安全需求，然后Codex就能协助你，将这些安全要求转化为具体的嵌入式C代码，包括故障诊断、降级模式等实现细节。

总的来说，这两个AI大模型，就像是为我们研发工程师量身定制的智能工具。它们不是要取代我们，而是要成为我们强大的助手，帮助我们提升效率、突破瓶颈，让我们能将更多的精力投入到创新和解决更深层次的工程挑战中去，无论是提升电机控制策略的性能，还是推动线控转向的未来发展。未来，AI在汽车电子和机器人执行器控制领域的应用潜力无限，值得我们持续关注和探索。

晨报 · AI赋能转向，机器人触觉与下一代汽车安全：研发工程师早报

2026-06-10
节目内容 / 应用分析

各位研发工程师朋友们，大家早上好！欢迎收听今天的科技早报。我是你们的主播兼技术分析师。今天，我们将聚焦AI前沿、汽车智能化和机器人这三大热门方向，精选出七条对我们工作最有启发性的资讯。从永磁同步电机的谐波抑制，到线控转向的安全挑战，再到机器人领域的接触智能和端到端控制，相信今天的节目会给大家带来不少思考和灵感。话不多说，让我们立刻进入今天的早报时间。

1. 离线强化学习在核聚变等高风险物理系统控制中的应用（来源：arXiv cs.LG）
应用分析：在我们的EPS系统研发中，在线路试和台架测试中积累了大量的运行数据。我们可以尝试将这些宝贵数据用于离线强化学习，来训练和优化电机控制策略，比如FOC矢量控制、无传感器控制，甚至弱磁控制等。比如，针对不同路况、不同驾驶风格下的手感调校，我们就可以通过离线强化学习，从历史数据中学习到最佳的助力曲线和阻尼补偿。这比我们传统基于模型的PID或者MPC控制，在适应性和鲁棒性上可能更有优势。我认为这是一个值得中长期投入的方向，有望提升我们的标定效率和系统性能。
https://arxiv.org/abs/2606.07550

2. 基于极值搜索控制的永磁同步电机位置传感器谐波自适应补偿（来源：arXiv eess.SY 控制系统）
应用分析：我们EPS系统广泛使用永磁同步电机，而位置传感器，无论是旋转变压器还是磁编码器，其安装误差或非线性特性都可能引入谐波。这项技术可以直接用于提升我们FOC控制的精度，减少转矩纹波和抖动，从而改善转向手感和降低NVH。它完全可以作为一个补充模块，集成到我们现有的电机控制软件中。这项研究的投入产出比非常高，我认为是值得我们立即研究和进行试验验证的，因为它能直接解决我们现在最想解决的问题之一。
https://arxiv.org/abs/2606.07906

3. 代理网络物理系统中的增益操纵攻击：稳定性与安全性的权衡（来源：arXiv eess.SY 控制系统）
应用分析：在我们的EPS系统向线控转向演进的过程中，网络安全和功能安全是生命线。AI或数据驱动的自适应控制策略，未来很可能涉及动态调整控制增益，这无疑会成为潜在的攻击点。我们需要设计更鲁棒的增益管理机制，例如，引入多层级的异常检测、增益范围约束，以及冗余验证机制，确保即使在增益被篡改的情况下，系统也能安全降级或保持基本功能。这应该在线控转向的架构设计阶段和ISO 26262功能安全流程中，作为高优先级问题进行深入分析和解决。
https://arxiv.org/abs/2606.07803

4. 超越灵巧：接触智能如何定义机器人技术的下一个时代（来源：IEEE Spectrum）
应用分析：我们可以借鉴机器人领域在“接触智能”上的前沿研究，来改进EPS的助力策略和手感调校。例如，通过更精密的力矩传感器，或者基于模型的状态观测器，我们能否更准确、更实时地感知驾驶员和路面之间微小的交互力？这将有助于我们实现更真实、更细腻的阻尼、回正和摩擦补偿，从而大幅提升路感还原能力。对于线控转向系统而言，由于完全没有机械连接，如何模拟最真实的物理反馈，接触智能更是其成功的关键。这是一个值得我们持续关注和探索的交叉点。
https://spectrum.ieee.org/agilink-contact-intelligence-robot-manipulation

5. 端到端控制的动力膝踝假肢：实现统一、免调校的辅助（来源：arXiv cs.RO）
应用分析：目前，EPS的助力策略调校仍然高度依赖大量人工经验和反复的路试，耗时耗力。借鉴这种端到端、免调校的思想，我们可以探索使用深度学习模型，直接从车辆状态、驾驶员输入、路面信息等多元传感器数据，输出最优的电机控制指令。这有望大幅提升我们的标定效率，并实现更智能、更个性化的助力手感。虽然这需要大量的驾驶数据和不同场景的覆盖，前期投入会比较大，但从长期来看，它能带来巨大的研发效率提升和用户体验优化，是值得我们深入研究和投入的。
https://arxiv.org/abs/2606.07902

6. EPC推出新型GaN基无刷直流电机逆变器板（来源：Power Electronics News）
应用分析：我们EPS的控制器功率级目前多采用硅基MOSFET或IGBT。GaN器件可以显著提高逆变器的效率，减小体积和重量，同时降低散热需求。这对于提升EPS的整体性能、在电动汽车中延长续航里程，以及实现更紧凑的系统设计至关重要。我们可以积极评估GaN器件在下一代EPS功率级中的应用潜力，从原型开发开始，验证其在汽车严苛环境下的可靠性和成本效益。这是我们关注的“功率级”和“芯片（功率器件）动态”的直接体现，值得我们投入资源进行前期研究和技术储备。
https://www.powerelectronicsnews.com/epc-launches-new-gan-based-bldc-inverter-boards/

7. 端到端AI在物理应用中未经充分审查的崛起令人担忧（来源：EE Times）
应用分析：在探索AI驱动的控制系统改进时，尤其是涉及线控转向等安全关键系统，我们必须警惕盲目采用端到端AI的风险。我们应该优先考虑结合传统控制理论和AI的混合架构，确保AI模块的可解释性、可验证性，并设计完善的故障诊断和降级方案。例如，AI可以用于辅助决策或优化参数，但最终的控制指令仍由经过严格验证的传统控制器生成。这需要我们投入大量精力在AI安全验证、形式化方法和功能安全标准制定上，确保AI的引入不会以牺牲安全性为代价。这是我们“用AI / 数据驱动方法改进控制”时必须面对和解决的核心问题。
https://www.eetimes.com/the-concerning-unchecked-rise-of-e2e-ai-in-physical-applications/
